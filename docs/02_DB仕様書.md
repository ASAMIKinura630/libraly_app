# 図書館保守システム — DB仕様書

## 1. 概要

| 項目 | 内容 |
|------|------|
| DBMS | PostgreSQL（Supabase ホスト） |
| スキーマ | `public`（デフォルト） |
| テーブル数 | 7 |
| テーブル名 | `libraly_app`（図書）、`libraly_app_category`（図書カテゴリ）、`libraly_app_faculty`（学部）、`libraly_app_department`（学科）、`libraly_app_student`（学生）、`libraly_app_staff`（担当者）、`libraly_app_lending_history`（図書貸し出し履歴） |
| RLS | **使用しない**（無効のまま運用） |
| 認証連携 | Supabase Auth（`auth.users`）を利用。図書・学生等のマスタに `user_id` は**持たない**。担当者マスタは `email` でログインと紐づけ、貸出履歴に処理者を記録する |
| ログイン UI | `index.html` のみ（別テーブル・別 HTML なし） |

## 3. テーブル定義

### 3.1 `libraly_app_category`（図書カテゴリマスタ）

ドロップダウンの選択肢となるカテゴリを管理する。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | カテゴリ識別子。自動採番 |
| `name` | `text` | `NOT NULL` / `UNIQUE` | — | カテゴリ名（一覧・ドロップダウン表示用） |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | マスタ登録日時 |

#### 補足

- カテゴリの CRUD は `categories.html`（マスタ保守ゾーン）から行う。初期データは SQL でも投入可能。
- 同名カテゴリの重複を防ぐため `name` に UNIQUE を付与する。

### 3.2 `libraly_app`（図書）

図書1件を1行で管理する。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | レコード識別子。自動採番 |
| `title` | `text` | `NOT NULL` | — | タイトル（必須） |
| `content` | `text` | `NOT NULL` | — | あらすじ（必須） |
| `name` | `text` | `NOT NULL` | — | 著者（必須） |
| `category_id` | `uuid` | `NULL` 可 / `FOREIGN KEY` | — | 図書カテゴリへの参照。**必須ではない** |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | 作成日時。INSERT 時に自動セット |

#### 外部キー

| 制約名 | 参照先 | ON DELETE |
|--------|--------|-----------|
| `libraly_app_category_id_fkey` | `libraly_app_category(id)` | `SET NULL` |

#### 補足

- 要求仕様上の「著者」は DB カラム名 `name` に対応する。
- 要求仕様上の「図書カテゴリ」は `category_id`（未設定時は `NULL`）。
- `id`・`created_at` はクライアントからの INSERT 時に省略可能（DB が生成）。

### 3.3 `libraly_app_faculty`（学部マスタ）

学部（親）を管理する。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | 学部識別子 |
| `name` | `text` | `NOT NULL` / `UNIQUE` | — | 学部名 |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | 登録日時 |

### 3.4 `libraly_app_department`（学科マスタ）

学科（子）を管理する。必ず1つの学部に所属する。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | 学科識別子 |
| `faculty_id` | `uuid` | `NOT NULL` / `FOREIGN KEY` | — | 所属学部（親） |
| `name` | `text` | `NOT NULL` | — | 学科名 |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | 登録日時 |

#### 外部キー・一意制約

| 制約名 | 内容 |
|--------|------|
| `libraly_app_department_faculty_id_fkey` | `faculty_id` → `libraly_app_faculty(id)` ON DELETE **RESTRICT** |
| `libraly_app_department_faculty_name_key` | 同一学部内で学科名は重複不可 `UNIQUE (faculty_id, name)` |

### 3.5 `libraly_app_student`（学生マスタ）

図書を借りる学生（利用者）1人を1行で管理する。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | レコード識別子 |
| `student_number` | `text` | `NOT NULL` / `UNIQUE` | — | 学籍番号 |
| `name` | `text` | `NOT NULL` | — | 氏名 |
| `department_id` | `uuid` | `NOT NULL` / `FOREIGN KEY` | — | 学科マスタへの参照 |
| `birth_date` | `text` | `NOT NULL` / CHECK | — | 生年月日（YYYYMMDD） |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | 登録日時 |

#### 外部キー

| 制約名 | 参照先 | ON DELETE |
|--------|--------|-----------|
| `libraly_app_student_department_id_fkey` | `libraly_app_department(id)` | **RESTRICT** |

#### 生年月日（`birth_date`）の制約

| ルール | 内容 |
|--------|------|
| 形式 | 8桁の数字文字列（例：`20000415`） |
| CHECK | `birth_date ~ '^\d{8}$'` かつ `to_date(birth_date, 'YYYYMMDD')` が有効な日付 |

#### 補足

- 学籍番号の重複を防ぐため `student_number` に UNIQUE を付与する。
- 表示時は `YYYYMMDD` を `YYYY/MM/DD` 等に変換してよい（DB には8桁文字列で保存）。
- 学部名は `department_id` → `libraly_app_department` → `libraly_app_faculty` の結合で取得する。
- 貸し出し履歴は `libraly_app_lending_history` で管理する（`3.6` 参照）。

### 3.6 `libraly_app_lending_history`（図書貸し出し履歴）

図書の貸出・返却の1回分を1行で管理する。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | 履歴レコード識別子（自動採番） |
| `book_id` | `uuid` | `NOT NULL` / `FOREIGN KEY` | — | 図書マスタのレコード識別子（`libraly_app.id`） |
| `student_number` | `text` | `NOT NULL` / `FOREIGN KEY` | — | 学生マスタの学籍番号 |
| `is_returned` | `boolean` | `NOT NULL` | `false` | **返却フラグ**（`false`=貸出中、`true`=返却済み） |
| `due_date` | `date` | `NOT NULL` | — | **返却予定日**（貸出登録時に設定） |
| `returned_at` | `timestamptz` | `NULL` 可 | `NULL` | **返却日時**（返却操作時に設定。未返却は `NULL`） |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | 貸出日時 |
| `lent_by_staff_id` | `uuid` | `NULL` 可 / `FOREIGN KEY` | `NULL` | **貸出処理者**（`libraly_app_staff.id`） |
| `returned_by_staff_id` | `uuid` | `NULL` 可 / `FOREIGN KEY` | `NULL` | **返却処理者**（`libraly_app_staff.id`） |
| `is_sub_lend` | `boolean` | `NOT NULL` | `false` | **また貸し記録済み**（公式借り手と実保持者が異なる） |
| `actual_holder_student_number` | `text` | `NULL` 可 / `FOREIGN KEY` | `NULL` | **実保持者**の学籍番号（公式借り手 `student_number` は変更しない） |
| `sub_lend_note` | `text` | `NULL` 可 | `NULL` | また貸しメモ（任意） |
| `sub_lend_recorded_at` | `timestamptz` | `NULL` 可 | `NULL` | また貸し記録日時 |
| `sub_lend_recorded_by_staff_id` | `uuid` | `NULL` 可 / `FOREIGN KEY` | `NULL` | **また貸し記録担当者**（`libraly_app_staff.id`） |

#### 外部キー

| 制約名 | 参照先 | ON DELETE |
|--------|--------|-----------|
| `libraly_app_lending_history_book_id_fkey` | `libraly_app(id)` | **RESTRICT** |
| `libraly_app_lending_history_student_number_fkey` | `libraly_app_student(student_number)` | **RESTRICT** |
| `libraly_app_lending_history_lent_by_staff_id_fkey` | `libraly_app_staff(id)` | **RESTRICT** |
| `libraly_app_lending_history_returned_by_staff_id_fkey` | `libraly_app_staff(id)` | **RESTRICT** |
| `libraly_app_lending_history_actual_holder_student_number_fkey` | `libraly_app_student(student_number)` | **RESTRICT** |
| `libraly_app_lending_history_sub_lend_recorded_by_staff_id_fkey` | `libraly_app_staff(id)` | **RESTRICT** |

#### 補足

- 図書の識別は **図書マスタの `id`**、学生の識別は **学籍番号 `student_number`** とする。
- 返却時は `is_returned` を `true`、`returned_at` を返却操作日時に更新する（`index.html` で操作）。
- 貸出・返却時、ログイン中のメールと一致する担当者を自動で `lent_by_staff_id` / `returned_by_staff_id` に記録する。
- カラム追加前の履歴は処理者が `NULL`（画面では「—」）。
- 同一図書の重複貸出可否はアプリ側で制御する（DB では履歴として複数行を許容）。
- 1人の学生が同時に借りられる冊数（**5冊まで**）もアプリ側で制御する（返却済みはカウントしない）。**公式借り手**としての未返却に加え、**また貸しの実保持者**としての未返却もカウントする。
- **また貸し**記録時は `is_sub_lend = true` とし、実保持者を `actual_holder_student_number` に保存する。公式借り手 `student_number` は変更しない。`is_sub_lend = true` のときは `actual_holder_student_number` が必須かつ `student_number` と異なる（CHECK 制約）。

### 3.6a `libraly_app_staff`（担当者マスタ）

貸出・返却の処理者（職員）を管理する。ログインアカウント（メール）と1対1で紐づける。

| カラム名 | 型 | 制約 | デフォルト | 説明 |
|----------|-----|------|------------|------|
| `id` | `uuid` | `PRIMARY KEY` | `gen_random_uuid()` | 担当者識別子 |
| `staff_number` | `text` | `NOT NULL` / `UNIQUE` | — | 職員番号 |
| `name` | `text` | `NOT NULL` | — | 氏名 |
| `email` | `text` | `NOT NULL` / `UNIQUE` | — | ログインメール（Supabase Auth と一致） |
| `created_at` | `timestamptz` | `NOT NULL` | `now()` | 登録日時 |

- 貸出履歴から参照されている担当者は削除不可（`ON DELETE RESTRICT`）。
- 新規登録（`index.html`）および担当者マスタ（`staff.html`）登録時は、入力氏名・メールで自動 insert する。職員番号は `STF-` + ランダム8桁で自動採番する。

## 4. DDL（作成・変更用 SQL）

```sql
CREATE TABLE public.libraly_app (
  id           uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  title        text        NOT NULL,
  content      text        NOT NULL,
  name         text        NOT NULL,
  category_id  uuid        NULL REFERENCES public.libraly_app_category(id) ON DELETE SET NULL,

```

### 4.3 学部・学科マスタ定義（参考）



### 5.2.1 図書一覧のカテゴリ絞り込み（アプリ層）

一覧見出し横の絞り込みでカテゴリを選択したときの表示制御。**DB スキーマの変更は不要**。

| 絞り込み | 表示対象 |
|----------|----------|
| すべて（未選択） | `category_id` を問わず全件 |
| 特定カテゴリを選択 | `category_id` が選択 ID と一致する行のみ |

実装例（クライアント側フィルタ）：

```javascript
const filtered = books.filter((b) => b.category_id === selectedCategoryId);
```

サーバー側で絞る場合の SQL 例：

```sql
SELECT b.*, c.name AS category_name
FROM public.libraly_app AS b
LEFT JOIN public.libraly_app_category AS c ON b.category_id = c.id
WHERE b.category_id = $1
ORDER BY b.created_at DESC;
```

### 5.3 図書登録（カテゴリ任意）

```sql
INSERT INTO public.libraly_app (title, content, name, category_id)
VALUES ($1, $2, $3, $4)
RETURNING *;
```

- `$4`（`category_id`）は未選択時 **NULL** を渡す。

### 5.4 図書更新

```sql
UPDATE public.libraly_app
SET title = $2, content = $3, name = $4, category_id = $5
WHERE id = $1
RETURNING *;
```

- カテゴリを外す場合は `category_id = NULL` を設定する。

## 7. 制約・バリデーション

| 層 | 内容 |
|----|------|
| DB | タイトル・あらすじ・著者は `NOT NULL`。`category_id` は **NULL 可** |
| DB | 存在しない `category_id` は外部キー制約で拒否 |
| アプリ | タイトル・あらすじ・著者の trim 後非空チェック |
| アプリ | カテゴリは未選択可。未選択時は `category_id` に `null` を送る |
| DB | 学生は `department_id` 必須。学部は学科経由で参照 |
| アプリ | 学部選択後に学科ドロップダウンを絞り込む |

## 8. セキュリティ方針

| 項目 | 方針 |
|------|------|
| RLS | 全業務テーブルで無効（`DISABLE ROW LEVEL SECURITY`） |
| `user_id` | 業務テーブルにカラムなし。担当者は `libraly_app_staff.email` で Auth と紐づけ |
| マスタ更新 | 図書・学生・カテゴリ・学部学科・担当者は各保守画面から CRUD。初期データは SQL 投入可 |
| リスク | anon キー漏洩時は業務データが CRUD 可能。認証は Supabase Auth で別管理 |
