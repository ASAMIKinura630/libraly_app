# 図書館保守システム — 要求仕様書

## 1. 概要

| 項目 | 内容 |
|------|------|
| システム名 | 図書館保守システム（libraly_app） |
| 目的 | 図書・学生等のマスタ保守、図書の貸出・返却、メール・パスワード認証を行う |
| データストア | Supabase（PostgreSQL） |
| 対象端末 | スマートフォン・PC（レスポンシブ対応） |

本ドキュメントは、図書リスト機能・図書カテゴリ管理機能・学生マスタ・ログイン機能の要求を定義する。

**メールアドレス・パスワードでのログイン機能を搭載している。**

## 2. スコープ

### 2.1 対象（In Scope）

- 図書の新規登録
- 図書情報の変更
- 図書の削除
- 登録済み図書の一覧表示
- 一覧からの編集・削除操作
- Supabase 上のテーブル `libraly_app` による図書データの永続化
- 図書カテゴリの設定（任意）および一覧でのカテゴリ名表示
- 図書カテゴリ選択肢のマスタテーブルによる管理
- 学生マスタ（図書貸出者）のデータ永続化
- 学生マスタの登録・一覧・編集・削除（`students.html`・保守ゾーン）
- 図書マスタの登録・一覧・編集・削除（`books.html`・保守ゾーン）
- 図書マスタでの貸出履歴照会（誰が・いつ借りたかの表示）
- 貸出履歴照会ポータル（図書起点・学生起点の体系化された照会）
- 図書カテゴリマスタの登録・一覧・編集・削除（`categories.html`・保守ゾーン）
- マスタ保守ポータル（`admin.html`）
- 学部マスタ・学科マスタ（親子関係）および学生からの参照
- 学部・学科マスタの登録・一覧・編集・削除（`masters.html`・保守ゾーン）
- 図書貸し出し履歴のデータ永続化（図書 `id`・学籍番号・返却フラグ）
- 貸出・返却画面（`index.html`・業務ゾーン・トップ）
- メールアドレス・パスワードによる認証（新規登録・ログイン・ログアウト・ログイン中メール表示）
- 認証 UI（`index.html` 内のログイン・新規登録）および共通認証モジュール（`js/auth.js`）

### 2.2 対象外（Out of Scope）

- OAuth・ソーシャルログイン・マジックリンク
- メール確認（confirm email）
- `user_id` による業務データの所有者管理（図書・学生等テーブルに `user_id` は持たない）
- Row Level Security（RLS）

## 3. 利用者・前提

| 項目 | 方針 |
|------|------|
| 利用者 | Supabase Auth に登録した利用者。未ログイン時は業務・保守画面へアクセスできない |
| 認証 | メールアドレス＋パスワード（Supabase Auth）。パスワードは6文字以上 |
| 権限分離 | 業務データの行単位分離は行わない（RLS 不使用・テーブルに `user_id` なし） |
| ネットワーク | Supabase に到達可能な環境（ブラウザから利用） |

## 4. 機能要求

### 4.1 図書登録（Create）

| ID | 要求 |
|----|------|
| FR-01 | タイトル・あらすじ・著者を入力して図書を登録できる |
| FR-02 | タイトル・あらすじ・著者はいずれも必須とする |
| FR-03 | 登録成功後、一覧に新しい図書が反映される |
| FR-04 | 必須項目未入力時は登録できず、エラーまたはバリデーション表示を行う |
| FR-04a | 図書登録時、図書カテゴリをドロップダウンから任意で選択できる |

### 4.2 図書一覧（Read）

| ID | 要求 |
|----|------|
| FR-05 | 登録済み図書を一覧で表示できる |
| FR-06 | 一覧には少なくともタイトル・著者・あらすじ（または要約）・作成日時を表示できる |
| FR-07 | 作成日時の新しい順（降順）で表示することを推奨 |
| FR-08 | 図書が0件の場合は、空状態であることが分かる表示とする |
| FR-08a | 一覧に、図書に設定されたカテゴリ名を表示する（未設定の場合は未設定であることが分かる表示とする） |
| FR-08b | 図書一覧の見出し（タイトル）の横で図書カテゴリを選択し、選択したカテゴリに属する図書のみ一覧に表示できる |
| FR-08c | 一覧のカテゴリ絞り込みで「すべて」を選んだときは、全図書を表示する |

### 4.3 図書変更（Update）

| ID | 要求 |
|----|------|
| FR-09 | 一覧から選択した図書のタイトル・あらすじ・著者を変更できる |
| FR-10 | 変更時もタイトル・あらすじ・著者は必須とする |
| FR-11 | 変更成功後、一覧の表示が更新される |
| FR-11a | 図書編集時、図書カテゴリをドロップダウンから任意で選択・変更できる |

### 4.4 図書削除（Delete）

| ID | 要求 |
|----|------|
| FR-12 | 一覧から選択した図書を削除できる |
| FR-13 | 誤削除防止のため、確認ダイアログ等の確認 UI を設けることを推奨 |
| FR-14 | 削除成功後、一覧から当該図書が消える |
| FR-15 | 図書一覧から貸出履歴画面（`book-history.html`）へ遷移し、その図書の貸出履歴（誰が・いつ借りたか）を表示できる |

### 4.5 カテゴリ管理機能

図書（本システムにおける1件の登録データ。以下「図書」）に、図書カテゴリを任意で紐づけられるようにする。

| ID | 要求 |
|----|------|
| FR-CAT-01 | 図書に図書カテゴリを設定できる |
| FR-CAT-02 | カテゴリの選択肢は、専用のマスタテーブルで管理する |
| FR-CAT-03 | 図書の新規登録・編集のとき、ドロップダウン（プルダウン）でカテゴリを選べる |
| FR-CAT-04 | 図書カテゴリの入力は必須項目としない（未選択で登録・更新できる） |
| FR-CAT-05 | 図書一覧画面に、各図書のカテゴリ名を表示する |
| FR-CAT-06 | ドロップダウンの選択肢は、マスタテーブルから取得した有効なカテゴリ一覧とする |
| FR-CAT-07 | 未選択時は、ドロップダウンに「未選択」等の空選択肢を用意し、一覧ではカテゴリ未設定と分かる表示とする |
| FR-CAT-08 | 図書一覧エリアの見出し横に、カテゴリ絞り込み用ドロップダウンを設ける |
| FR-CAT-09 | 絞り込みでカテゴリを選択した場合、その `category_id` を持つ図書のみ一覧に表示する |
| FR-CAT-10 | カテゴリマスタを `categories.html` から登録・一覧・編集・削除できる |

#### 補足

- マスタテーブルへの初期データ投入（SQL）は DB 側で行う。カテゴリは画面からも CRUD 可能（`categories.html`）。
- 図書テーブルとマスタテーブルの物理定義は `02_DB仕様書.md` で定義する。

### 4.6 学生マスタ

図書を借りる学生（利用者）の情報を、学生マスタで管理する。

| ID | 要求 |
|----|------|
| FR-STU-01 | 学籍番号・氏名・学部・学科・生年月日を保存できる |
| FR-STU-02 | 学籍番号は重複しない |
| FR-STU-03 | 生年月日は **YYYYMMDD** 形式（8桁数字）で保存する |
| FR-STU-04 | 生年月日は有効な日付であること（例：`20000415`） |
| FR-STU-05 | データは専用テーブル `libraly_app_student` に永続化する |
| FR-STU-06 | 学部・学科は **学部マスタ・学科マスタ** を参照する（テキスト直入力しない） |
| FR-STU-07 | 学科は所属学部（親）に紐づく。フォームでは学部選択後に学科を選ぶ |
| FR-STU-08 | 学生一覧から貸出履歴画面（`student-history.html`）へ遷移し、その学生の貸出履歴（何を・いつ借りたか）を表示できる |

#### 補足

- 本フェーズは DB テーブル定義と **学生マスタ画面（`students.html`）** による CRUD。
- 学部・学科マスタの画面 CRUD（`masters.html`）

### 4.6b 担当者マスタ

貸出・返却の処理者（職員）を管理する。

| ID | 要求 |
|----|------|
| FR-STAFF-01 | 氏名・ログインメールを `libraly_app_staff` に保存できる（職員番号は自動採番） |
| FR-STAFF-02 | 職員番号・メールアドレスは重複しない |
| FR-STAFF-03 | `staff.html` で担当者の登録・一覧・編集・削除ができる |
| FR-STAFF-04 | ログインメールと一致する担当者がいない場合、貸出・返却はできない |
| FR-STAFF-05 | 職員番号は `STF-` + ランダム8桁で自動採番する（新規登録・担当者マスタ登録とも） |
| FR-STAFF-06 | 新規登録時に担当者 insert が失敗した場合、ログイン時に Auth の氏名メタデータから再試行する |

### 4.6a ナビゲーション・画面体系

| ID | 要求 |
|----|------|
| FR-NAV-01 | ログイン済み画面にグローバルナビ（貸出・返却／貸出履歴照会／マスタ保守）を表示する |
| FR-NAV-02 | 貸出履歴照会ポータル（`inquiry.html`）から図書起点・学生起点の照会に遷移できる |
| FR-NAV-03 | 照会モード（`?mode=inquiry`）では参照のみとし、マスタの登録・編集・削除を非表示とする |
| FR-NAV-04 | 履歴詳細画面にパンくずで階層（業務区分・照会経路）を示す |

### 4.7 貸出・返却

図書と学生を紐づけて貸し出し履歴を管理し、返却を記録する。

| ID | 要求 |
|----|------|
| FR-LEND-01 | 図書（マスタの `id`）と学生（学籍番号）を選択して貸出を登録できる |
| FR-LEND-02 | 貸出登録時、返却フラグは `false`（貸出中）とする |
| FR-LEND-03 | 貸出中の図書は、同一図書の再貸出を登録できない |
| FR-LEND-03a | 1人の学生が**同時に**借りられる図書は **5冊まで** とする（返却済みはカウントしない）。**公式借り手**（`student_number`）としての貸出に加え、**また貸しの実保持者**（`actual_holder_student_number`）として未返却の履歴もカウントする |
| FR-LEND-03b | 6冊目の貸出登録は拒否し、エラーメッセージを表示する |
| FR-LEND-04 | 貸し出し履歴一覧を表示できる（図書名・著者・氏名・**実保持者**・貸し出し日・返却予定日・返却日・貸出処理者・状態）。**返却処理者**は DB に記録するが、業務画面（`index.html`）の一覧列には表示しない |
| FR-LEND-05 | 貸出中の履歴に対して返却操作ができ、返却フラグを `true` にし返却日を記録する |
| FR-LEND-05b | 貸出登録時、ログイン担当者を貸出処理者として自動記録する |
| FR-LEND-05c | 返却時、ログイン担当者を返却処理者として自動記録する |
| FR-LEND-05a | 貸出登録時、貸出日は当日・返却予定日は14日後を自動設定する |
| FR-LEND-06 | 返却前に確認ダイアログを表示する |
| FR-LEND-07 | 状態（すべて / 貸出中 / **延滞** / **また貸し** / 返却済み）およびキーワードで履歴を絞り込める |
| FR-LEND-07a | **延滞**とは、未返却（`is_returned = false`）かつ返却予定日（`due_date`）が今日より前の履歴 |
| FR-LEND-07b | 貸出・返却画面から **延滞リスト（CSV）** をダウンロードできる（延滞判定は FR-LEND-07a と同一） |
| FR-LEND-07c | **返却予定が近い**とは、未返却・未延滞かつ返却予定日が **今日から3日以内** の履歴。一覧絞り込みと **返却予定リスト（CSV）** で出力できる |
| FR-LEND-07d | 延滞履歴には **延滞日数**（返却予定日からの経過日数）を表示する。延滞 CSV にも含める |
| FR-LEND-07e | ログイン後の貸出・返却画面に **貸出状況サマリー**（貸出中件数・延滞件数・返却予定が近い件数・**また貸し件数**）を表示する |
| FR-LEND-08 | データは `libraly_app_lending_history` に永続化する |
| FR-LEND-09a | **また貸し**（公式借り手と異なる学生が実際に本を保持している状態）を、貸出中の履歴に対して記録できる |
| FR-LEND-09b | また貸し記録時、**公式借り手**（`student_number`）は変更しない |
| FR-LEND-09c | また貸し記録時、**実保持者**（`actual_holder_student_number`）を指定する。公式借り手と同一の学生は指定不可 |
| FR-LEND-09d | 実保持者も FR-LEND-03a の **5冊上限** のカウント対象とする |
| FR-LEND-09e | **延滞判定・延滞リスト・督促対象**は公式借り手基準のままとする（また貸し記録では変更しない） |
| FR-LEND-09f | また貸し記録時、記録日時・記録担当者・任意メモを保存する |
| FR-LEND-09g | **また貸しリスト（CSV）** をダウンロードできる（貸出中かつ `is_sub_lend = true` の全件） |
| FR-LEND-09h | また貸し記録時、実保持者の選択は **学部・学科** で絞り込める（貸出登録の学生指定と同様） |

#### 補足

- 画面は **`index.html`**（トップ）で実装する。旧 `lending.html` はリダイレクト。
- 図書・学生マスタにデータがない場合は、適宜メッセージを表示する。

### 4.8 認証（ログイン）

Supabase Auth により、メールアドレスとパスワードでシステムへ入る。業務データテーブルへの `user_id` 付与は行わない。

| ID | 要求 |
|----|------|
| FR-AUTH-01 | `index.html` の新規登録タブでメールアドレス・氏名・パスワードを登録できる |
| FR-AUTH-01a | 新規登録成功時、入力したメール・氏名で担当者マスタ（`libraly_app_staff`）に自動登録する |
| FR-AUTH-02 | `index.html` のログインタブでメールアドレス・パスワードでログインできる |
| FR-AUTH-03 | ログイン中、各業務・保守画面のヘッダー付近にログイン担当者の氏名を表示できる（担当者未登録時はメールアドレス） |
| FR-AUTH-04 | ログアウト操作でセッションを破棄し、`index.html` のログイン／新規登録表示に戻れる |
| FR-AUTH-05 | パスワードは6文字以上とする（クライアント・Supabase 双方で検証） |
| FR-AUTH-06 | メール確認（confirm email）は行わない（Supabase ダッシュボードで無効化） |
| FR-AUTH-07 | 新規登録・ログイン成功後は同一 `index.html` 上で貸出・返却画面を表示する |
| FR-AUTH-08 | 認証エラーは `alert()` で表示する |
| FR-AUTH-09 | 未ログインで業務・保守画面にアクセスした場合は `index.html` へリダイレクトする |

#### 補足

- 認証方式はメール＋パスワードのみ（OAuth・マジックリンクは対象外）。
- 実装の詳細は `03_画面仕様書.md`・`04_機能仕様書.md` を参照。

## 6. 非機能要求

### 6.1 デバイス・UI

| ID | 要求 |
|----|------|
| NFR-01 | スマートフォン・PC の両方で操作可能であること |
| NFR-02 | 画面幅に応じたレイアウト調整（レスポンシブ）を行うこと |
| NFR-03 | 主要操作（登録・一覧・編集・削除）がタッチ操作でも問題なく行えること |

### 6.2 データ・セキュリティ

| ID | 要求 |
|----|------|
| NFR-04 | 図書は `libraly_app`、カテゴリマスタ・学生マスタは専用テーブルに Supabase 上で保存する |
| NFR-05 | RLS は有効化しない |
| NFR-06 | `user_id` カラムは持たない |
| NFR-07 | Supabase の anon キー等を用いる場合、テーブルへの CRUD 権限はポリシーではなく API 設定・GRANT で制御する（RLS 非使用の前提） |

### 6.3 可用性・運用

| ID | 要求 |
|----|------|
| NFR-08 | Supabase 障害時は、ユーザーにエラーが伝わること |
| NFR-09 | 通信中はローディング等で二重送信を抑止することを推奨 |

## 7. 画面構成（現行実装）

グローバルナビで **業務・照会・マスタ保守** の3区分。認証 UI は `index.html` のみ。

```
【index.html — 未ログイン】
  ログイン / 新規登録（メール・氏名・パスワード）

【index.html — ログイン後（業務）】
  認証バー（担当者氏名）+ グローバルナビ
  貸出登録 / 貸し出し履歴・返却

【inquiry.html → 照会モード】
  books.html?mode=inquiry / students.html?mode=inquiry
  → book-history.html / student-history.html（パンくず）

【admin.html → 各マスタ画面】
  認証バー + グローバルナビ + 保守サブナビ
  books / categories / students / staff / masters
```

図書の登録・編集・削除は **`books.html`**（マスタ保守ゾーン）で行う。機能画面は計10（`lending.html` は `index.html` へリダイレクト）。

**プレゼンテーション（説明用・認証不要）:** `presentation/` 配下に概要・画面説明スライド・ERD を掲載する。5分プレゼン台本（`5min.html`）は直接 URL のみとし、他ページからはリンクしない（`03_画面仕様書.md` §16 参照）。

## 8. API・連携（方針）

フロントエンドから Supabase Client を用い、以下の操作を行う。

| 操作 | Supabase API（例） |
|------|-------------------|
| 図書一覧取得 | `select` on `libraly_app`（カテゴリ名はマスタと結合または別取得） |
| 図書登録 | `insert` into `libraly_app` |
| 図書変更 | `update` on `libraly_app` by `id` |
| 図書削除 | `delete` on `libraly_app` by `id` |
| カテゴリ選択肢取得 | `select` on カテゴリマスタテーブル |
| 学生マスタ CRUD | `select` / `insert` / `update` / `delete` on `libraly_app_student` |
| 新規登録 | `auth.signUp`（email / password）→ `libraly_app_staff` に insert（氏名・メール） |
| ログイン | `auth.signInWithPassword` |
| ログアウト | `auth.signOut` |
| セッション確認 | `auth.getSession` |

## 9. 受入基準

### 9.1 図書 CRUD（`books.html`）

- [ ] `books.html` でタイトル・あらすじ・著者を入力して図書を登録できる
- [ ] `books.html` の「貸出履歴」から `book-history.html` に遷移し、図書情報と貸出者・貸し出し日等の履歴が表示される
- [ ] 登録した図書が一覧に表示される
- [ ] 一覧から図書を編集し、内容が保存・反映される
- [ ] 一覧から図書を削除し、一覧から消える
- [ ] スマートフォン・PC のブラウザで上記が操作できる
- [ ] `libraly_app` テーブルにデータが永続化される
- [ ] RLS が無効であり、`user_id` カラムが存在しない

### 9.2 カテゴリ管理

- [ ] 図書の新規登録・編集時に、マスタ由来のカテゴリをドロップダウンで選択できる
- [ ] カテゴリ未選択のまま図書を登録・更新できる
- [ ] カテゴリを設定した図書は、一覧にカテゴリ名が表示される
- [ ] カテゴリ未設定の図書は、一覧で未設定と分かる
- [ ] カテゴリ選択肢がマスタテーブルの内容と一致する
- [ ] 一覧見出し横のカテゴリ選択で、該当カテゴリの図書のみ表示される
- [ ] 「すべて」選択時は全図書が表示される

### 9.3 学生マスタ

- [ ] `students.html` で学生を登録・一覧・編集・削除できる
- [ ] 学籍番号・氏名・学部・学科・生年月日（YYYYMMDD）を保存できる
- [ ] 学籍番号が重複しない
- [ ] 不正な生年月日（8桁以外・存在しない日付）は拒否される
- [ ] 学部・学科はマスタからドロップダウンで選択する
- [ ] `students.html` の「貸出履歴」から `student-history.html` に遷移し、学生情報と貸出履歴が表示される

### 9.4 認証

- [ ] `index.html` の新規登録でメール・氏名・パスワード（6文字以上）を登録できる
- [ ] 新規登録後、担当者マスタに同名・同メールの担当者が自動登録される
- [ ] `index.html` のログインで認証でき、成功後に貸出・返却画面が表示される
- [ ] ログイン中、画面に担当者氏名が表示される
- [ ] ログアウト後、`index.html` のログイン表示に戻る
- [ ] 未ログインで `admin.html` 等に直接アクセスすると `index.html` へ誘導される
- [ ] 認証エラーは `alert()` で表示される

### 9.5 貸出・返却（`index.html`）

- [ ] 図書と学生を選択して貸出登録でき、貸出日（当日）・返却予定日（14日後）が自動設定される
- [ ] 履歴一覧に返却予定日・返却日が表示される
- [ ] 貸出中の同一図書の再貸出はできない
- [ ] 1人の学生が同時に5冊までしか借りられず、6冊目の貸出はエラー表示される
- [ ] 履歴一覧で返却操作ができる（確認ダイアログあり）
- [ ] 状態・キーワードで履歴を絞り込める（延滞のみで返却予定日超過の未返却を表示できる）
- [ ] 「延滞リスト(CSV)」ボタンで延滞中の履歴を CSV ファイルとしてダウンロードできる
- [ ] 「返却予定が近い」絞り込みと「返却予定リスト(CSV)」で3日以内の未返却を確認・出力できる
- [ ] 履歴一覧に延滞日数（または期限間近の残り日数）が表示される
- [ ] 貸出状況サマリーに貸出中・延滞・返却予定が近い件数が表示される

## 10. 用語

| 用語 | 説明 |
|------|------|
| 図書 | 本システムで管理する1冊（1レコード）の情報単位 |
| 図書カテゴリ | 図書に任意で付与する分類。選択肢はマスタテーブルで管理 |
| カテゴリマスタ | 図書カテゴリの選択肢を定義する専用テーブル |
| 学生マスタ | 図書を借りる学生の情報を管理するテーブル |
| RLS | Row Level Security。行単位のアクセス制御。本システムでは使用しない |
| Supabase | BaaS。PostgreSQL データベースと API を提供する |
