## 3. ゾーン構成

| ゾーン | 画面 | 利用頻度 | 利用者の意識 |
|--------|------|----------|--------------|
| 業務 | SCR-OP-01 | 高 | 図書を貸す・返す（未ログイン時は同画面でログイン） |
| 照会 | SCR-INQ-00 ほか | 中 | 貸出履歴を参照する（操作はしない） |
| マスタ保守 | SCR-MT-00〜05 | 低 | マスタデータを慎重に直す |
| 学生 | SCR-STU-01 | 高（学生） | 自分で図書を借りる（職員ナビ・保守画面は表示しない） |

**原則:** 職員向け画面ではグローバルナビで3区分（貸出・返却／貸出履歴照会／マスタ保守）を表示する。`index.html` で未ログインのときは `#auth-gate` のみ表示し、ナビは出さない。学生向け `student-borrow.html` ではグローバルナビ・マスタ保守・職員画面へのリンクを出さない。

## 5. ナビゲーション仕様

### 5.0 認証バー（業務・保守ゾーン共通）

ログイン済みの業務・保守画面では、`.app` 先頭に認証バー（`#auth-bar`）を表示する。

| 要素 | 仕様 |
|------|------|
| 表示内容 | `ログイン中:` ＋ 担当者氏名（未登録時はメールアドレス） |
| ログアウト | 「ログアウト」ボタン。押下で `auth.signOut` 後 `index.html` のログイン表示へ |
| スタイル | `css/auth-bar.css` |
| 実装 | `js/auth.js` の `LibralyAuth.renderUserBar` / `requireAuth` |

### 5.1 グローバルナビ（全画面共通）

ログイン済み画面では、認証バー直下に **グローバルナビ**（`#global-nav`）を表示する。

| 表示テキスト | リンク先 | 区分 |
|--------------|----------|------|
| 貸出・返却 | `index.html` | 業務 |
| 貸出履歴照会 | `inquiry.html` | 照会 |
| マスタ保守 | `admin.html` | 保守 |

- スタイル: `css/global-nav.css`（`.global-nav`）
- 実装: `js/nav.js` の `LibralyNav.renderGlobalNav`
- 現在の区分は `aria-current="page"` で強調表示する
- `aria-label`: `メインメニュー`

### 5.2 共通ルール

| # | ルール |
|---|--------|
| N-01 | グローバルナビは全ログイン済み画面で同一の3項目とする |
| N-02 | 現在ページ・区分は `aria-current="page"` を付与する |
| N-03 | マスタ保守ゾーンではグローバルナビの下に **サブナビ** を表示する |
| N-04 | 照会ゾーンの詳細画面では **パンくず** で階層を示す |
| N-05 | 「← 貸出・返却に戻る」は廃止し、グローバルナビで代替する |

### 5.3 マスタ保守サブナビ（SCR-MT-00〜05）

マスタ保守区分の画面では、グローバルナビ直下にサブナビ（`#sub-nav`）を表示する。

```
マスタ保守 ｜ 図書 ｜ カテゴリ ｜ 学生 ｜ 担当者 ｜ 学部・学科
```

| 表示テキスト | リンク先 |
|--------------|----------|
| マスタ保守 | `admin.html` |
| 図書 | `books.html` |
| カテゴリ | `categories.html` |
| 学生 | `students.html` |
| 担当者 | `staff.html` |
| 学部・学科 | `masters.html` |

- `aria-label`: `マスタ保守メニュー`

### 5.4 照会ゾーン（SCR-INQ-00 ほか）

| 画面 | グローバルナビ | サブナビ | パンくず |
|------|----------------|----------|----------|

## 6.1 認証 UI（SCR-OP-01 内・未ログイン時）

**実装ファイル:** `index.html`（`#auth-gate`）

### 目的

メールアドレス・パスワードでログインまたは新規登録し、**ページ遷移なし**で `#app-main`（貸出・返却 UI）を表示する。別ファイルのログイン画面は使わない。

未ログイン時、`#auth-gate` 内に「学生の方：セルフ貸出はこちら」リンク（`student-borrow.html`）を表示する。職員ログイン画面に学生／職員の切替タブは設けない（入口 URL で分ける）。

学生アカウント（`user_metadata.role === 'student'`）でログインした場合、または既存の学生セッションがある場合は `student-borrow.html` へリダイレクトし、職員 UI は表示しない。

### DOM 構成

| 要素 ID | 表示タイミング |
|---------|----------------|
| `#auth-gate` | 未ログイン時のみ |
| `#app-main` | ログイン後のみ（ナビ・貸出・履歴を含む） |
| `#auth-bar` | `#app-main` 内。ログイン担当者名・ログアウト |

### 操作

| 要素 | 仕様 |
|------|------|
| タブ | 「ログイン」「新規登録」（`#auth-tab-login` / `#auth-tab-signup`）。`aria-selected` で状態管理 |
| ログイン | `signInWithPassword`。成功 → `showAppMain()` で `#app-main` 表示・データ読込 |
| 新規登録 | `signUpWithStaff`（メール・氏名・パスワード）。成功時に担当者マスタへ自動登録 → 同上 |
| ログアウト | `signOut` 後 `index.html` を再読込（`#auth-gate` 表示） |
| エラー | `alert()`（`LibralyAuth.authErrorMessage`） |
| タイトル | 未ログイン: `ログイン \| 図書館保守システム` / ログイン後: `貸出・返却 \| …` |

### 入力項目

**ログイン**

| 項目 | 必須 | 備考 |
|------|------|------|
| メールアドレス | ○ | `type="email"` |
| パスワード | ○ | 6文字以上 |

**新規登録**

| 項目 | 必須 | 備考 |
|------|------|------|
| メールアドレス | ○ | `type="email"`。担当者マスタのログインメールにも使用 |
| 氏名 | ○ | 担当者マスタの氏名にも使用 |
| パスワード | ○ | 6文字以上 |

新規登録成功時、職員番号（`STF-` + ランダム8桁）・氏名・メールで `libraly_app_staff` に自動登録する。登録失敗時はログイン後に再試行する。

### Supabase 設定（運用）

Authentication → Providers → Email で **Confirm email を無効** にすること。

## 7. 貸出・返却画面（SCR-OP-01）

**実装ファイル:** `index.html`

### 7.1 画面の目的

図書と学生を選択して貸出を登録し、貸出中の履歴を返却する。履歴の参照・絞り込みも行う。あわせて **LINE 返却リマインド** の送信オン/オフを設定する。

### 7.1a LINE 返却リマインド設定（ログイン後）

| 項目 | 内容 |
|------|------|
| 位置 | `#app-main` 内、サマリーの直上 |
| デフォルト | **オフ**（`libraly_app_settings.line_reminder_enabled = false`） |
| UI | チェックボックス「返却リマインドの自動送信を有効にする」＋状態表示 |
| オン時確認 | Messaging API 送信・課金の注意ダイアログ |
| 変更権限 | ログイン中かつ担当者マスタ登録済みの職員のみ |
| 送信予定 | 毎日 **JST 20:00** — 返却3日前 / 返却前日 / 延滞 |
| 本番開始 | オン切替と cron 設定は `docs/12_LINE返却リマインド本番開始手順.md` を参照 |

### 7.2 レイアウト

**未ログイン（`#auth-gate`）**

```

│  h1: 図書館保守システム                    │
│  [ログイン] [新規登録] タブ                 │
│  ログイン: メール / パスワード              │
│  新規登録: メール / 氏名 / パスワード        │
└──────────────────────────────────────────┘
```

**ログイン後（`#app-main`）**

```
┌──────────────────────────────────────────┐
│  [認証バー] ログイン中: 山田 太郎 [ログアウト] │
│  [ナビ] 貸出・返却｜貸出履歴照会｜マスタ保守 │
├──────────────────────────────────────────┤
│  h1 / サブタイトル                         │
├──────────────────────────────────────────┤
```

## 7a. 学生セルフ貸出画面（SCR-STU-01）

**実装ファイル:** `student-borrow.html`  
**対象:** 学生（スマートフォン想定）。担当者業務画面（SCR-OP-01）とは **別 URL・別ゾーン**（統合ログインは行わない）。

### 7a.1 画面の目的

学生が自分でログインし、図書に貼付した QR コードをスキャンして貸出を登録する。返却・また貸しは担当者画面で行う（本画面では不可）。ログイン後は **貸出処理に必要な UI のみ** を表示する。

### 7a.2 未ログイン時（認証ゲート）

#### LINE アプリ内（LIFF）

| ブロック | 内容 |
|----------|------|
| `#line-loading` | 「LINE でログインしています…」 |
| `#line-init-error` | LIFF 初期化・ID トークンエラー（赤メッセージ） |
| `#line-link-panel` | 初回紐づけフォーム（見出し: 「学籍番号と LINE を紐づけ」）。学籍番号・生年月日（8桁） |
| `#line-browser-hint` | LINE 外ブラウザ直開き時の案内（リッチメニューから開くよう促す） |

LIFF 設定済みかつ LINE クライアント内では、上記を優先表示し、パスワード認証は `#password-auth-section` に残す（LINE 不調時の予備）。

#### パスワード認証（予備）

| タブ | 入力 | 動作 |
|------|------|------|
| ログイン | 学籍番号・パスワード | Supabase Auth（`{学籍番号}@libraly.student`）でサインイン |
| 初回登録 | 学籍番号・生年月日（8桁）・パスワード | 学生マスタ照合後にアカウント作成、`auth_user_id` を紐づけ |

職員向け画面・マスタ保守へのリンクは **表示しない**（学生用 URL を直接案内する）。

### 7a.3 ログイン後

| ブロック | 内容 |
|----------|------|
| 見出し | 「学生セルフ貸出」＋説明文 |
| 学生バー | 氏名・学籍番号・（LINE ログイン時は `/ LINE` 表示）・貸出中冊数（公式借り手＋また貸し実保持者を含む上限 5 冊）・ログアウト |
| 図書を借りる | `html5-qrcode` でカメラ起動 → QR 読取 → 図書プレビュー →「この図書を借りる」 |
| あなたの貸出中 | 未返却履歴の一覧（タイトル・返却期限） |

グローバルナビ・マスタ保守・照会・職員トップへの導線は出さない。

### 7a.4 QR コード仕様

- ペイロード: `LIBRALY_BOOK:{図書UUID}`（`js/book-qr.js`）
- 生成（一覧）: 図書マスタ保守（SCR-MT-01）の各行「QRコード」ボタン → モーダル表示
- 生成（詳細）: 図書貸出履歴（SCR-MT-01-HIST）の図書情報内に常時表示（保守モードのみ）・「QR を印刷」
- ライブラリ: CDN `qrcode@1.4.4`（`QRCode.toCanvas`）

### 7a.5 貸出登録ルール

| 項目 | 値 |
|------|-----|
| `student_number` | ログイン中の学生 |
| `lent_by_staff_id` | NULL |
| `is_self_service` | true |
| `due_date` | 貸出日から 14 日 |
| 上限・重複 | 業務画面と同様（5 冊上限・貸出中の図書は再貸出不可） |

## 16. プレゼンテーション — 概要ページ（SCR-PRES-01）

### 16.2 データ入力ルール（表示文言）

「ログイン保護と担当者記録」の**直前**に見出し **データ入力ルール** を表示する。

| 見出し（h3） | 説明 |
|--------------|------|
| 図書はカテゴリマスタを参照 | 図書カテゴリマスタからドロップダウンで選択（未設定可）。自由入力テキストでは分類しない |

| 同時貸出は5冊まで | 1学生が**同時に**借りられる図書は**5冊まで**（返却済みはカウントしない）。公式借り手・また貸し実保持者の両方をカウント |
| また貸しの記録 | 公式借り手（A）と異なる学生（B）が本を保持する場合に記録。Aは変更せず、延滞・督促はA基準 |

### 16.3 プレゼンサイト共通ナビ

`presentation/overview.html`・`index.html`・`erd.html`・`5min.html` 等で共通の sticky ナビ（`css/present-nav.css`）を表示する。

**実装ファイル:** `presentation/5min.html`・`presentation/script-5min.md`
