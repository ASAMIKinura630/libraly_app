| 項目 | 内容 |

## 2. 開発前提（画面関連）

| 項目 | 方針 |
|------|------|
| 技術 | HTML / CSS / JavaScript のみ |
| フレームワーク | React / Vue / Next.js は使わない |
| パッケージ管理 | npm は使わない |
| Supabase SDK | CDN から読み込む |
| 共通モジュール | `js/auth.js`（Supabase クライアント・認証）、`css/auth-bar.css`（認証バー） |
| 画面数 | 6（`index.html` / `admin.html` / `books.html` / `categories.html` / `students.html` / `masters.html`） |
| リダイレクト | `lending.html` → `index.html`（即時リダイレクト） |
| ログイン | Supabase Auth（メール＋パスワード）。**`index.html` のみ**（`#auth-gate` 内タブ切替） |
| 未ログイン時 | `index.html` は `#auth-gate` を表示。他画面は `index.html` へ `location.replace` |
| ログイン後 | `index.html` は `#app-main`（貸出・返却）。他画面は `#auth-bar` ＋ 各機能 UI |
| RLS | 使用しない |
| user_id | 使わない |
| 対応端末 | スマートフォン・PC の両方で見やすくする |

## 3. ゾーン構成

| ゾーン | 画面 | 利用頻度 | 利用者の意識 |
|--------|------|----------|--------------|
| 業務 | SCR-OP-01 | 高 | 図書を貸す・返す（未ログイン時は同画面でログイン） |
| マスタ保守 | SCR-MT-00〜04 | 低 | マスタデータを慎重に直す |

**原則:** 業務ゾーンのナビにマスタ CRUD を同列で並べない。保守ゾーンに入ったら業務メニューと保守メニューを混在させない。`index.html` で未ログインのときは `#auth-gate` のみ表示し、業務ナビ・保守ナビは出さない。

## 5. ナビゲーション仕様

### 5.0 認証バー（業務・保守ゾーン共通）

ログイン済みの業務・保守画面では、`.app` 先頭に認証バー（`#auth-bar`）を表示する。

| 要素 | 仕様 |
|------|------|
| 表示内容 | `ログイン中:` ＋ メールアドレス |
| ログアウト | 「ログアウト」ボタン。押下で `auth.signOut` 後 `index.html` のログイン表示へ |
| スタイル | `css/auth-bar.css` |
| 実装 | `js/auth.js` の `LibralyAuth.renderUserBar` / `requireAuth` |

### 5.1 共通ルール

| # | ルール |
|---|--------|
| N-01 | ナビは `<nav>` でマークアップし、適切な `aria-label` を付与する |
| N-02 | 現在ページは `aria-current="page"` を付与する |
| N-03 | 業務ゾーンと保守ゾーンでナビ項目セットを変える |
| N-04 | 保守ゾーン全画面に「← 貸出・返却に戻る」（`index.html`）を置く |
| N-05 | 業務ゾーンには「マスタ保守 →」（`admin.html`）を1つだけ置く |

### 5.2 業務ゾーン（SCR-OP-01）

```
貸出・返却                              マスタ保守 →
```

| 表示テキスト | リンク先 | 備考 |
|--------------|----------|------|
| 貸出・返却 | `index.html` | 現在ページ |
| マスタ保守 → | `admin.html` | 右寄せ（`.nav__admin-link`） |

- `aria-label`: `メインメニュー`
- 図書・学生・学部学科への直接リンクは **表示しない**

### 5.3 マスタ保守ゾーン（SCR-MT-00〜04）

```
← 貸出・返却に戻る

マスタ保守 ｜ 図書 ｜ カテゴリ ｜ 学生 ｜ 学部・学科
```

| 表示テキスト | リンク先 |
|--------------|----------|
| ← 貸出・返却に戻る | `index.html` |
| マスタ保守 | `admin.html` |
| 図書 | `books.html` |
| カテゴリ | `categories.html` |
| 学生 | `students.html` |
| 学部・学科 | `masters.html` |

- 復帰リンク: `aria-label="業務画面への復帰"`
- 保守メニュー: `aria-label="マスタ保守メニュー"`

### 5.4 保守バナー（全保守画面）

ヘッダー直前に `.maintenance-banner`（`role="note"`）を固定表示する。

| 画面 | バナー文言 |
|------|------------|
| SCR-MT-00 | マスタ保守モード — 登録データの追加・変更・削除はこのエリアで行います |
| SCR-MT-01 | マスタ保守モード — 変更内容は貸出画面の図書選択肢に反映されます |
| SCR-MT-02 | マスタ保守モード — 学籍番号は貸出履歴と紐づきます。変更・削除は慎重に行ってください |
| SCR-MT-04 | マスタ保守モード — 変更内容は図書マスタのカテゴリ選択肢に反映されます |
| SCR-MT-03 | マスタ保守モード — 学科の変更・削除は学生マスタに影響します |

## 6.1 認証 UI（SCR-OP-01 内・未ログイン時）

**実装ファイル:** `index.html`（`#auth-gate`）

### 目的

メールアドレス・パスワードでログインまたは新規登録し、**ページ遷移なし**で `#app-main`（貸出・返却 UI）を表示する。別ファイルのログイン画面は使わない。

### DOM 構成

| 要素 ID | 表示タイミング |
|---------|----------------|
| `#auth-gate` | 未ログイン時のみ |
| `#app-main` | ログイン後のみ（ナビ・貸出・履歴を含む） |
| `#auth-bar` | `#app-main` 内。ログイン中メール・ログアウト |

### 操作

| 要素 | 仕様 |
|------|------|
| タブ | 「ログイン」「新規登録」（`#auth-tab-login` / `#auth-tab-signup`）。`aria-selected` で状態管理 |
| ログイン | `signInWithPassword`。成功 → `showAppMain()` で `#app-main` 表示・データ読込 |
| 新規登録 | `signUp`。成功 → 同上 |
| ログアウト | `signOut` 後 `index.html` を再読込（`#auth-gate` 表示） |
| エラー | `alert()`（`LibralyAuth.authErrorMessage`） |
| タイトル | 未ログイン: `ログイン \| 図書館保守システム` / ログイン後: `貸出・返却 \| …` |

### 入力項目

| 項目 | 必須 | 備考 |
|------|------|------|
| メールアドレス | ○ | `type="email"` |
| パスワード | ○ | 6文字以上 |

### Supabase 設定（運用）

Authentication → Providers → Email で **Confirm email を無効** にすること。

## 7. 貸出・返却画面（SCR-OP-01）

**実装ファイル:** `index.html`

### 7.1 画面の目的

図書と学生を選択して貸出を登録し、貸出中の履歴を返却する。履歴の参照・絞り込みも行う。

### 7.2 レイアウト

**未ログイン（`#auth-gate`）**

```
┌──────────────────────────────────────────┐
│  h1: 図書館保守システム                    │
│  [ログイン] [新規登録] タブ                 │
│  メール / パスワード / [ログイン or 登録]    │
└──────────────────────────────────────────┘
```

**ログイン後（`#app-main`）**

```
┌──────────────────────────────────────────┐
│  [認証バー] ログイン中: user@example.com [ログアウト] │
│  [ナビ] 貸出・返却          マスタ保守 →   │
├──────────────────────────────────────────┤
│  h1 / サブタイトル                         │
├──────────────────────────────────────────┤
│  メッセージエリア                           │
├──────────────────────────────────────────┤
│  【貸出登録】  ┌図書を指定┐ ┌学生を指定┐    │
│              │カテゴリ▼ │ │学部▼…   │    │
│              │図書▼     │ │学生▼    │    │
│              └─────────┘ └─────────┘    │
│              [貸出する]                   │
├──────────────────────────────────────────┤
│  【貸し出し履歴】 状態▼                      │
│    ┌図書で検索┐ ┌学生で検索┐               │
│    │図書名/著者│ │氏名      │               │
│    └─────────┘ └─────────┘               │
│  テーブル（状態/図書/著者/学籍/氏名/日時/操作）│
└──────────────────────────────────────────┘
```

### 7.3 主要要素

| 要素 | 仕様 |
|------|------|
| 貸出登録見出し | `h2`「貸出登録」の横に補足文「条件を選択して図書と学生を指定してください」（`.card-header__hint`・ muted 色） |
| 図書・学生のレイアウト | 「図書を指定」「学生を指定」を別パネル表示。**スマホ: 縦並び** / **600px以上: 横並び** |
| 図書カテゴリ絞り込み | `libraly_app_category` から取得。「すべて」「未設定」＋各カテゴリ。選択に応じて図書ドロップダウンを絞り込む |
| 図書ドロップダウン | `libraly_app` から取得。表示形式: `図書名（カテゴリ）（著者）`。貸出中の図書は `disabled` |
| 学部絞り込み | `libraly_app_faculty` から取得。「すべて」＋各学部。選択に応じて学科・学生を絞り込む |
| 学科絞り込み | 学部未選択時は無効。学部選択後 `libraly_app_department` から「すべて」＋該当学科を表示し、学生を絞り込む |
| 学生ドロップダウン | `libraly_app_student` から取得。表示形式: `学籍番号 — 氏名（学部 / 学科）` |
| 貸出ボタン | 「貸出する」。`is_returned: false` で INSERT |
| 状態絞り込み | すべて / 貸出中のみ / 返却済みのみ。**初期値: 貸出中のみ** |
| 図書で検索 | 図書名・著者を別パネル表示。各欄部分一致（AND）。**スマホ: 縦並び** / **600px以上: 横並び**（学生で検索と並列） |
| 学生で検索 | 氏名を別パネル表示。部分一致。図書検索条件と AND |
| 返却ボタン | 貸出中の行のみ。「返却」→ 確認後 `is_returned: true` |

### 7.4 空状態・警告メッセージ

| 状況 | 文言 |
|------|------|
| 履歴0件 | 貸し出し履歴がありません。上のフォームから貸出を登録してください。 |
| 絞り込み0件 | 条件に一致する履歴がありません。絞り込みや検索条件を変更してください。 |
| 図書0件 | 図書が登録されていません。マスタ保守の図書マスタで登録してください。 |
| 学生0件 | 学生が登録されていません。マスタ保守の学生マスタで登録してください。 |

### 7.5 返却確認ダイアログ

```
「{図書タイトル}」を {氏名} さんから返却しますか？
```

`libraly_app_lending_history` / `libraly_app` / `libraly_app_student` / `libraly_app_category` / `libraly_app_faculty` / `libraly_app_department`

## 12. 学部・学科マスタ保守画面（SCR-MT-03）

**実装ファイル:** `masters.html`

### 12.1 画面の目的

### 12.2 構成

## 13. 共通 UI 仕様

### 13.1 メッセージ表示

| 種別 | CSS クラス | 用途 |
|------|------------|------|
| 成功 | `message--success` | 保存・返却成功等 |
| 警告 | `message--warn` | バリデーション・編集中 |
