## 2. 開発前提（画面関連）

| 項目 | 方針 |
|------|------|
| 技術 | HTML / CSS / JavaScript のみ |
| フレームワーク | React / Vue / Next.js は使わない |
| パッケージ管理 | npm は使わない |
| Supabase SDK | CDN から読み込む |
| 共通モジュール | `js/auth.js`（認証）、`css/theme.css`（共通テーマ）、`css/auth-bar.css`（認証バー）、`css/global-nav.css`（ナビ） |
| 画面数 | 機能画面10 + リダイレクト1（`index.html` / `inquiry.html` / `admin.html` / `books.html` / `book-history.html` / `students.html` / `student-history.html` / `categories.html` / `masters.html` / `staff.html` / `lending.html`） |
| リダイレクト | `lending.html` → `index.html`（即時リダイレクト） |
| ログイン | Supabase Auth（メール＋パスワード）。**`index.html` のみ**（`#auth-gate` 内タブ切替） |
| 未ログイン時 | `index.html` は `#auth-gate` を表示。他画面は `index.html` へ `location.replace` |
| ログイン後 | `index.html` は `#app-main`（貸出・返却）。他画面は `#auth-bar` ＋ 各機能 UI |
| RLS | 使用しない |
| user_id | 使わない |
| 対応端末 | スマートフォン・PC の両方で見やすくする |

## 5. ナビゲーション仕様

### 5.0 認証バー（業務・保守ゾーン共通）

ログイン済みの業務・保守画面では、`.app` 先頭に認証バー（`#auth-bar`）を表示する。

| 要素 | 仕様 |
|------|------|
| 表示内容 | `ログイン中:` ＋ 担当者氏名（未登録時はメールアドレス） |
| ログアウト | 「ログアウト」ボタン。押下で `auth.signOut` 後 `index.html` のログイン表示へ |
| スタイル | `css/auth-bar.css` |
| 実装 | `js/auth.js` の `LibralyAuth.renderUserBar` / `requireAuth` |

### 5.1 グローバルナビ（全画面共通）

ログイン済み画面では、認証バー直下に **グローバルナビ**（`#global-nav`）を表示する。

| 表示テキスト | リンク先 | 区分 |
|--------------|----------|------|
| 貸出・返却 | `index.html` | 業務 |
| 貸出履歴照会 | `inquiry.html` | 照会 |
| マスタ保守 | `admin.html` | 保守 |

- スタイル: `css/global-nav.css`（`.global-nav`）
- 実装: `js/nav.js` の `LibralyNav.renderGlobalNav`
- 現在の区分は `aria-current="page"` で強調表示する
- `aria-label`: `メインメニュー`

### 5.2 共通ルール

| # | ルール |
|---|--------|
| N-01 | グローバルナビは全ログイン済み画面で同一の3項目とする |
| N-02 | 現在ページ・区分は `aria-current="page"` を付与する |
| N-03 | マスタ保守ゾーンではグローバルナビの下に **サブナビ** を表示する |
| N-04 | 照会ゾーンの詳細画面では **パンくず** で階層を示す |
| N-05 | 「← 貸出・返却に戻る」は廃止し、グローバルナビで代替する |

### 5.3 マスタ保守サブナビ（SCR-MT-00〜05）

マスタ保守区分の画面では、グローバルナビ直下にサブナビ（`#sub-nav`）を表示する。

```
マスタ保守 ｜ 図書 ｜ カテゴリ ｜ 学生 ｜ 担当者 ｜ 学部・学科
```

| 表示テキスト | リンク先 |
|--------------|----------|
| マスタ保守 | `admin.html` |
| 図書 | `books.html` |
| カテゴリ | `categories.html` |
| 学生 | `students.html` |
| 担当者 | `staff.html` |
| 学部・学科 | `masters.html` |

- `aria-label`: `マスタ保守メニュー`
- 実装: `LibralyNav.renderMaintenanceSubNav`

### 5.4 照会ゾーン（SCR-INQ-00 ほか）

| 画面 | グローバルナビ | サブナビ | パンくず |
|------|----------------|----------|----------|
| `inquiry.html` | 貸出履歴照会が現在地 | なし | なし |
| `books.html?mode=inquiry` | 貸出履歴照会が現在地 | なし | 貸出・返却 › 貸出履歴照会 › 図書から照会 |
| `students.html?mode=inquiry` | 貸出履歴照会が現在地 | なし | 貸出・返却 › 貸出履歴照会 › 学生から照会 |

照会モードでは `.inquiry-banner` を表示し、登録フォーム・編集・削除操作は非表示とする。

### 5.5 パンくず（履歴詳細画面）

`book-history.html` / `student-history.html` では、グローバルナビの下にパンくず（`#breadcrumb`）を表示する。

- 照会経路（`from=inquiry`）: 貸出・返却 › 貸出履歴照会 › 図書（学生）から照会 › 現在ページ
- 保守経路: 貸出・返却 › マスタ保守 › 図書（学生）マスタ › 現在ページ（サブナビも表示）

実装: `LibralyNav.renderBreadcrumb`

## 6.1 認証 UI（SCR-OP-01 内・未ログイン時）

**実装ファイル:** `index.html`（`#auth-gate`）

### 目的

メールアドレス・パスワードでログインまたは新規登録し、**ページ遷移なし**で `#app-main`（貸出・返却 UI）を表示する。別ファイルのログイン画面は使わない。

### DOM 構成

| 要素 ID | 表示タイミング |
|---------|----------------|
| `#auth-gate` | 未ログイン時のみ |
| `#app-main` | ログイン後のみ（ナビ・貸出・履歴を含む） |
| `#auth-bar` | `#app-main` 内。ログイン担当者名・ログアウト |

### 操作

| 要素 | 仕様 |
|------|------|
| タブ | 「ログイン」「新規登録」（`#auth-tab-login` / `#auth-tab-signup`）。`aria-selected` で状態管理 |
| ログイン | `signInWithPassword`。成功 → `showAppMain()` で `#app-main` 表示・データ読込 |
| 新規登録 | `signUpWithStaff`（メール・氏名・パスワード）。成功時に担当者マスタへ自動登録 → 同上 |
| ログアウト | `signOut` 後 `index.html` を再読込（`#auth-gate` 表示） |
| エラー | `alert()`（`LibralyAuth.authErrorMessage`） |
| タイトル | 未ログイン: `ログイン \| 図書館保守システム` / ログイン後: `貸出・返却 \| …` |

### 入力項目

**ログイン**

| 項目 | 必須 | 備考 |
|------|------|------|
| メールアドレス | ○ | `type="email"` |
| パスワード | ○ | 6文字以上 |

**新規登録**

| 項目 | 必須 | 備考 |
|------|------|------|
| メールアドレス | ○ | `type="email"`。担当者マスタのログインメールにも使用 |
| 氏名 | ○ | 担当者マスタの氏名にも使用 |
| パスワード | ○ | 6文字以上 |

新規登録成功時、職員番号（`STF-` + ランダム8桁）・氏名・メールで `libraly_app_staff` に自動登録する。登録失敗時はログイン後に再試行する。

### Supabase 設定（運用）

Authentication → Providers → Email で **Confirm email を無効** にすること。

## 7. 貸出・返却画面（SCR-OP-01）

**実装ファイル:** `index.html`

### 7.1 画面の目的

図書と学生を選択して貸出を登録し、貸出中の履歴を返却する。履歴の参照・絞り込みも行う。

### 7.2 レイアウト

**未ログイン（`#auth-gate`）**

```
┌──────────────────────────────────────────┐
│  h1: 図書館保守システム                    │
│  [ログイン] [新規登録] タブ                 │
│  ログイン: メール / パスワード              │
│  新規登録: メール / 氏名 / パスワード        │
└──────────────────────────────────────────┘
```

**ログイン後（`#app-main`）**

```
┌──────────────────────────────────────────┐
│  [認証バー] ログイン中: 山田 太郎 [ログアウト] │
│  [ナビ] 貸出・返却          マスタ保守 →   │
├──────────────────────────────────────────┤
│  h1 / サブタイトル                         │
├──────────────────────────────────────────┤
│  メッセージエリア                           │
├──────────────────────────────────────────┤
│  【貸出登録】  ┌図書を指定┐ ┌学生を指定┐    │
│              │カテゴリ▼ │ │学部▼…   │    │
│              │図書▼     │ │学生▼    │    │
│              └─────────┘ └─────────┘    │
│              [貸出する]                   │
├──────────────────────────────────────────┤
│  【貸し出し履歴】 状態▼                      │
│    ┌図書で検索┐ ┌学生で検索┐               │
│    │図書名/著者│ │氏名      │               │
│    └─────────┘ └─────────┘               │
│  テーブル（状態/図書/著者/学籍/氏名/貸し出し日/返却予定日/返却日/操作）│
└──────────────────────────────────────────┘
```

### 7.3 主要要素

| 要素 | 仕様 |
|------|------|
| 貸出登録見出し | `h2`「貸出登録」の横に補足文「条件を選択して図書と学生を指定してください」（`.card-header__hint`・ muted 色） |
| 図書・学生のレイアウト | 「図書を指定」「学生を指定」を別パネル表示。**スマホ: 縦並び** / **600px以上: 横並び** |
| 図書カテゴリ絞り込み | `libraly_app_category` から取得。「すべて」「未設定」＋各カテゴリ。選択に応じて図書ドロップダウンを絞り込む |
| 図書ドロップダウン | `libraly_app` から取得。表示形式: `図書名（カテゴリ）（著者）`。貸出中の図書は `disabled` |
| 学部絞り込み | `libraly_app_faculty` から取得。「すべて」＋各学部。選択に応じて学科・学生を絞り込む |
| 学科絞り込み | 学部未選択時は無効。学部選択後 `libraly_app_department` から「すべて」＋該当学科を表示し、学生を絞り込む |
| 学生ドロップダウン | `libraly_app_student` から取得。表示形式: `学籍番号 — 氏名（学部 / 学科）` |
| 学生パネルヒント | 1人の学生が同時に借りられる図書は5冊まで |
| 貸出ボタン | 「貸出する」。`created_at`=当日・`due_date`=14日後・`is_returned: false` で INSERT（日付は自動設定）。**貸出中が5冊の学生には6冊目を貸し出せない** |
| 状態絞り込み | すべて / 貸出中のみ / 返却済みのみ。**初期値: 貸出中のみ** |
| 図書で検索 | 図書名・著者を別パネル表示。各欄部分一致（AND）。**スマホ: 縦並び** / **600px以上: 横並び**（学生で検索と並列） |
| 学生で検索 | 氏名を別パネル表示。部分一致。図書検索条件と AND |
| 履歴列 | 状態・図書・著者・学籍番号・氏名・**貸し出し日**（日付のみ）・**返却予定日**・**返却日**・**貸出処理者**・**返却処理者**・操作 |
| 担当者 | ログインに紐づく担当者を自動記録。未登録時は貸出・返却不可 |
| 返却ボタン | 貸出中の行のみ。「返却」→ 確認後 `is_returned: true`・`returned_at` を当日に設定 |

### 7.4 空状態・警告メッセージ

| 状況 | 文言 |
|------|------|
| 履歴0件 | 貸し出し履歴がありません。上のフォームから貸出を登録してください。 |
| 絞り込み0件 | 条件に一致する履歴がありません。絞り込みや検索条件を変更してください。 |
| 図書0件 | 図書が登録されていません。マスタ保守の図書マスタで登録してください。 |
| 学生0件 | 学生が登録されていません。マスタ保守の学生マスタで登録してください。 |
| 貸出上限超過 | この学生はすでに5冊貸出中です。返却してから再度貸出してください。 |


## 15. 共通 UI 仕様

### 15.1 メッセージ表示

| 種別 | CSS クラス | 用途 |
|------|------------|------|
| 成功 | `message--success` | 保存・返却成功等 |
