## 4. 共通仕様

### 4.1 Supabase クライアント

- ページ読み込み時にクライアントを1つ初期化する
- URL・anon key は実装時に定数で保持（本書には記載しない）

### 4.2 バリデーション

| チェック | 条件 | 失敗時 |
|----------|------|--------|
| タイトル | `trim()` 後、長さ > 0 | F-05 入力エラー |
| あらすじ | 同上 | 同上 |
| 著者 | 同上 | 同上 |
| 図書カテゴリ | **チェックしない**（未選択可） | — |

### 4.3 編集モード

| 変数 | 意味 |
|------|------|
| `editingId` | 編集中図書の `id`。新規時は `null` |
| `listFilterCategoryId` | 一覧絞り込み中のカテゴリ `id`。全件表示時は `null` または `""` |

### 4.4 カテゴリ ID の扱い

| ドロップダウン値 | 送る `category_id` |
|------------------|-------------------|
| 「未選択」（`value=""`） | `null` |
| マスタの `id` | その uuid |

## 5. 各機能の詳細

### 5.1 図書新規保存機能（F-01）

#### 目的

タイトル・あらすじ・著者と、任意の図書カテゴリを新規登録する。

#### 入力値

| 入力名 | 取得元 | DBカラム |
|--------|--------|----------|
| タイトル | テキスト入力 | `title` |
| あらすじ | テキストエリア | `content` |
| 著者 | テキスト入力 | `name` |
| 図書カテゴリ | ドロップダウン | `category_id`（未選択時 `null`） |

#### 処理の流れ

| 順番 | 処理 |
|------|------|
| 1 | ユーザーがフォームに入力（カテゴリは任意） |
| 2 | 「保存」押下 |
| 3 | `editingId` が無いことを確認 |
| 4 | タイトル・あらすじ・著者のバリデーション |
| 5 | `libraly_app` に **insert**（`category_id` 含む） |
| 6 | 成功時：フォームクリア（カテゴリは「未選択」に戻す） |
| 7 | F-02 で一覧再取得 |
| 8 | F-05 で成功メッセージ |

#### Supabase 操作

```javascript
.from('libraly_app').insert({
  title, content, name,
  category_id: categoryId || null
})
```

---

### 5.2 図書一覧表示機能（F-02）

#### 目的

登録済み図書を、**カテゴリ名付き**で新しい順に一覧表示する。

#### 処理の流れ

| 順番 | 処理 |
|------|------|
| 1 | 起動時および CRUD 成功後に実行 |
| 2 | `libraly_app` を select（カテゴリ名を結合取得） |
| 3 | `created_at` 降順 |
| 4 | 各件にカテゴリ名を表示（未設定は「未設定」） |
| 5 | 編集・削除ボタンを付与 |
| 6 | F-07 の絞り込み条件を適用してから描画する |

#### Supabase 操作

```javascript
.from('libraly_app')
.select('id, title, content, name, category_id, created_at, libraly_app_category ( name )')
.order('created_at', { ascending: false })
```

#### 出力・表示内容

| 表示項目 | 備考 |
|----------|------|
| カテゴリ名 | 結合結果または `category_id` からマスタ照合 |
| タイトル | — |
| 著者 | — |
| あらすじ | — |
| 作成日時 | 日本語フォーマット |
| 編集・削除 | `id` を F-03 / F-04 へ |

---

### 5.3 図書編集機能（F-03）

#### 目的

既存図書のタイトル・あらすじ・著者・図書カテゴリを更新する。

#### 処理の流れ

| 順番 | 処理 |
|------|------|
| 1 | 「編集」押下 |
| 2 | フォームに `title`・`content`・`name`・`category_id` をセット |
| 3 | `editingId` を保持 |
| 4 | ユーザーが修正し「更新」押下 |
| 5 | 必須3項目のバリデーション |
| 6 | **update**（`category_id` は null も可） |
| 7 | フォームクリア・`editingId` 解除 |
| 8 | F-02・F-05 |

#### Supabase 操作

```javascript
.from('libraly_app').update({
  title, content, name,
  category_id: categoryId || null
}).eq('id', editingId)
```

### 5.5 メッセージ表示機能（F-05）

| 種別 | 例 |
|------|-----|
| 保存成功 | 図書を保存しました |

### 5.6 カテゴリ選択肢取得機能（F-06）

#### 目的

投稿・編集フォームのドロップダウンに、DB のマスタ一覧を表示する。

#### 要求仕様との対応

| 要求ID | 対応 |
|--------|------|
| FR-CAT-02 | マスタテーブル `libraly_app_category` から取得 |
| FR-CAT-03 | 新規・編集フォームのドロップダウンに反映 |
| FR-CAT-06 | 有効なマスタ行をすべて選択肢にする |
| FR-CAT-07 | 先頭に「未選択」を固定で置く |

#### 処理の流れ

| 順番 | 処理 |
|------|------|
| 1 | ページ読み込み時に実行 |
| 2 | `libraly_app_category` から `id`・`name` を **select** |
| 3 | `name` 昇順で並べる |
| 4 | `<select>` を再構築：先頭 `未選択` + 各カテゴリ |
| 5 | 編集復帰時は以前の選択値を維持 |

#### Supabase 操作

```javascript
.from('libraly_app_category')
.select('id, name')
.order('name', { ascending: true })
```

#### エラー時

- F-05 でエラー表示
- ドロップダウンは「未選択」のみ（図書の保存はカテゴリなしで可能）

---

### 5.7 一覧カテゴリ絞り込み機能（F-07）

#### 目的

図書一覧の見出し横でカテゴリを選び、**そのカテゴリに属する図書だけ**を一覧に表示する。

#### 入力値

| 入力名 | 取得元 |
|--------|--------|
| 表示カテゴリ | 一覧見出し横の `<select>` |

#### 処理の流れ

| 順番 | 処理 |
|------|------|
| 1 | F-06 と同様のマスタで絞り込み用 `<select>` を構築（先頭は「すべて」） |
| 2 | ユーザーがカテゴリを変更 |
| 3 | `listFilterCategoryId` を更新 |
| 4 | `booksCache` から `category_id` が一致する行だけ抽出 |
| 5 | 「すべて」のときは全件を表示 |
| 6 | F-02 の描画処理でフィルタ後の配列を表示 |

#### 正常時の動き

- カテゴリ変更直後に一覧が切り替わる（DB 再取得は不要）
- CRUD 後も絞り込み状態を維持する

#### 0件時

- フィルタ結果が0件なら「このカテゴリの図書はありません」等を表示
