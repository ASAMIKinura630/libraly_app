## 2. 開発前提（機能関連）

| 項目 | 方針 |
|------|------|
| 言語 | HTML / CSS / JavaScript のみ |
| Supabase SDK | CDN（`@supabase/supabase-js` v2） |
| クライアント | `LibralyAuth.getClient()`（`js/auth.js`）で全画面共有 |
| 認証 | Supabase Auth（メール＋パスワード）。UI は **`index.html` のみ** |
| RLS | 使用しない |
| user_id | 業務テーブルには持たない |
| エラー（認証） | `alert()` |
| エラー（業務） | 各画面の `#message` または `alert()`（返却確認は `confirm()`） |

**用語の対応**

| 呼び方 | 意味 | DB |
|--------|------|-----|
| 図書 | 1件の登録データ | `libraly_app` |
| 図書カテゴリ | 図書に任意で付与する分類 | `category_id` → `libraly_app_category` |
| 学生 | 貸出者 | `libraly_app_student` |
| 貸し出し履歴 | 図書と学生の貸出・返却記録 | `libraly_app_lending_history` |
| 担当者 | 貸出・返却の処理者（職員） | `libraly_app_staff` |

## 4. 共通仕様

### 4.1 Supabase クライアント

- `js/auth.js` の `LibralyAuth.getClient()` で 1 インスタンスを共有
- URL・anon key は `js/auth.js` に定数（本書には記載しない）

### 4.2 認証フロー

| 画面 | 起動時 |
|------|--------|
| `index.html` | `getSession()` → なし: `#auth-gate` / あり: `#app-main` + 業務データ読込 |
| その他 | `requireAuth()` → 失敗時 `index.html` へ `replace` → `renderUserBar('#auth-bar')` |

ログイン・新規登録成功時は **別 URL へ遷移せず** `index.html` 上で `#app-main` を表示する。

### 4.3 通信中

- 主要ボタンを `disabled` にし、二重送信を抑止（`setBusy` 等）

## 5. 認証機能（F-AUTH-01〜05）

### 5.1 新規登録（F-AUTH-01）

| 項目 | 内容 |
|------|------|
| UI | `index.html` の「新規登録」タブ（`#signup-form`） |
| 入力 | メールアドレス・氏名・パスワード |
| API | `LibralyAuth.signUpWithStaff({ email, password, name })` |
| 処理 | 1. `auth.signUp` 2. `libraly_app_staff` に insert（職員番号はメールローカル部から自動採番） |
| 成功 | `showAppMain()` — 貸出・返却 UI 表示・マスタ読込 |
| 失敗 | `alert(LibralyAuth.authErrorMessage(error))` |
| 部分失敗 | Auth 成功・担当者 insert 失敗時はエラー表示（手動で担当者マスタ登録を案内） |

### 5.2 ログイン（F-AUTH-02）

| 項目 | 内容 |
|------|------|
| UI | `index.html` の「ログイン」タブ（`#login-form`） |
| API | `auth.signInWithPassword({ email, password })` |
| 成功 | F-AUTH-01 と同様 |
| 失敗 | `alert()` |

### 5.3 ログアウト（F-AUTH-03）

| 項目 | 内容 |
|------|------|
| トリガー | `#auth-bar` の「ログアウト」 |
| API | `auth.signOut()` |
| 成功 | `location.replace('index.html')` — 未ログイン表示 |

### 5.4 ログイン状態表示（F-AUTH-04）

- 担当者マスタの氏名を「ログイン中:」の右に表示（未登録時はメールアドレス）
- 対象: `index.html`（`#app-main` 内）および全保守画面

### 5.5 未ログインガード（F-AUTH-05）

以下の全画面で `LibralyAuth.requireAuth()` を必須とする（`index.html` の `#app-main` 表示前も `getSession` で判定）。

| 区分 | ファイル |
|------|----------|
| 業務 | `index.html`（ログイン後 UI） |
| 照会 | `inquiry.html` / `books.html` / `students.html` / `book-history.html` / `student-history.html` |
| 保守 | `admin.html` / `books.html` / `categories.html` / `students.html` / `staff.html` / `masters.html` |

- セッションなし → `index.html` へ `location.replace`
- `lending.html` は認証なしで `index.html` へリダイレクト

### 5.6 バリデーション（認証）

| チェック | 条件 | 失敗時 |
|----------|------|--------|
| メール | `trim()` 後、長さ > 0 | `alert` |
| 氏名（新規登録のみ） | `trim()` 後、長さ > 0 | `alert`（氏名を入力してください。） |
| パスワード | 長さ ≥ 6 | `alert`（パスワードは6文字以上で入力してください。） |

## 6. 貸出・返却機能（F-LEND-01〜09）

**実装:** `index.html`（`#app-main` 内）

### 6.1 貸出登録（F-LEND-01）

| 順番 | 処理 |
|------|------|
| 1 | 図書・学生をドロップダウンで選択 |
| 2 | 「貸出する」押下 |
| 3 | `getCurrentStaff()` でログイン担当者を取得。未登録ならエラー表示して中断 |
| 4 | `libraly_app_lending_history` に `insert`（`created_at`=当日・`due_date`=14日後・`is_returned: false`・`lent_by_staff_id`=担当者 ID） |
| 5 | 履歴・図書選択肢を再描画 |

### 6.2 貸出中図書の再貸出防止（F-LEND-04）

- 履歴キャッシュから `is_returned === false` の `book_id` を収集
- 図書ドロップダウンで該当図書を `disabled`

### 6.3 返却（F-LEND-03）

- 貸出中の行のみ「返却」ボタン
- `getCurrentStaff()` で担当者を確認。未登録ならエラー表示して中断
- `confirm()` で確認後、`is_returned: true`・`returned_at` を当日・`returned_by_staff_id` を担当者 ID に `update`

### 6.4 履歴表示・絞り込み（F-LEND-02, F-LEND-05, F-LEND-06）

| 機能 | 内容 |
|------|------|
| 一覧 | 状態・図書名・著者・学籍番号・氏名・貸し出し日（日付のみ）・返却予定日・返却日・貸出処理者・返却処理者・返却状態 |
| 状態絞り込み | すべて / 貸出中のみ / 返却済みのみ（初期: 貸出中のみ） |
| キーワード | 図書名・著者・氏名の部分一致（条件は AND） |

### 6.5 貸出フォームの絞り込み（F-LEND-07）

- 図書: カテゴリでドロップダウン絞り込み
- 学生: 学部 → 学科 → 学生の順で絞り込み

## 7. 図書マスタ機能（F-01〜07）

**実装:** `books.html`

### 7.1 図書新規保存（F-01）

タイトル・あらすじ・著者と、任意の図書カテゴリを新規登録する。

| 入力名 | DBカラム |
|--------|----------|
| タイトル | `title` |
| あらすじ | `content` |
| 著者 | `name` |
| 図書カテゴリ | `category_id`（未選択時 `null`） |

```javascript
.from('libraly_app').insert({
  title, content, name,
  category_id: categoryId || null
})
```

### 7.2 図書一覧表示（F-02）

- `libraly_app` を select（カテゴリ名結合）、`created_at` 降順
- 未設定カテゴリは「未設定」表示
- F-07 の絞り込み後に描画

```javascript
.from('libraly_app')
.select('id, title, content, name, category_id, created_at, libraly_app_category ( name )')
.order('created_at', { ascending: false })
```

### 7.3 図書編集（F-03）

- 「編集」でフォームに値をセット、`editingId` を保持
- 「更新」で `update`（`category_id` は null 可）

### 7.4 図書削除（F-04）

- 確認後 `delete`
- 貸出履歴がある図書は DB 制約で削除不可（エラー表示）

### 7.5 メッセージ表示（F-05）

| 種別 | 例 |
|------|-----|
| 成功 | 図書を保存しました / 更新しました / 削除しました |
| エラー | 必須入力エラー、通信失敗 |

### 7.6 カテゴリ選択肢取得（F-06）

- `libraly_app_category` から `name` 昇順
- フォーム先頭は「未選択」

### 7.7 一覧カテゴリ絞り込み（F-07）

- 見出し横の select でクライアント側フィルタ
- 「すべて」で全件

### 7.8 図書貸出履歴照会（F-08）

**実装:** `book-history.html`（入口は `books.html` の「貸出履歴」リンク）

| 項目 | 内容 |
|------|------|
| トリガー | `books.html` 一覧の「貸出履歴」リンク（`?id={図書UUID}`） |
| ヘッダー | `libraly_app` を `id` で取得（カテゴリ結合） |
| 明細 | `libraly_app_lending_history` を `book_id` で絞り込み、学生名を結合 |
| 表示 | 状態・学籍番号・氏名・貸し出し日・返却予定日・返却日・貸出処理者・返却処理者 |
| 並び | `created_at` 降順 |

```javascript
.from('libraly_app_lending_history')
.select('id, student_number, is_returned, due_date, returned_at, created_at, libraly_app_student ( name ), lent_staff:libraly_app_staff!lent_by_staff_id ( name ), returned_staff:libraly_app_staff!returned_by_staff_id ( name )')
.eq('book_id', bookId)
.order('created_at', { ascending: false })
```

## 11. ページ読み込み時の実行順

### index.html

```
1. getSession()
2a. 未ログイン → #auth-gate 表示（終了）
2b. ログイン済 → showAppMain() → renderUserBar → 担当者取得 → 図書・学生・履歴等を並列 load
```

### 保守画面（例: books.html）

```
1. requireAuth()
2. renderUserBar
3. loadCategories → loadBooks
4. イベント設定
```

## 14. ナビゲーション（F-NAV-01〜04）

### 14.1 グローバルナビ（F-NAV-01）

全ログイン済み画面の認証バー直下に3項目を表示する。

### 14.2 照会ポータル（F-NAV-02）

### 14.3 照会モード（F-NAV-03）

| 画面 | 照会モード時の変更 |

| `students.html` | 同上 |

`book-history.html` / `student-history.html` で、照会経路（`from=inquiry`）または保守経路に応じたパンくずを表示する。
