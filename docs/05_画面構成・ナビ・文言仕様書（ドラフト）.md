| 誤操作の心理的ハードルを上げる | 保守画面では文言・配色・ナビで「保守モード」であることを明示する |

## 2. ゾーン構成

アプリ全体を **2ゾーン** に分ける。

```
┌─────────────────────────────────────────┐
│  業務ゾーン（Operations）                  │
│  ・貸出・返却（トップ）                     │
└─────────────────────────────────────────┘
              │
              │  「マスタ保守」入口（1リンク）
              ▼
┌─────────────────────────────────────────┐
│  マスタ保守ゾーン（Master Maintenance）    │
│  ・保守ポータル                            │
│  ・図書マスタ / 学生マスタ / 学部・学科     │
└─────────────────────────────────────────┘
```

| ゾーン | 利用頻度 | 利用者の意識 |
|--------|----------|--------------|
| 業務ゾーン | 高（毎日） | 「図書を貸す・返す」 |
| マスタ保守ゾーン | 低（必要時のみ） | 「マスタデータを直す。慎重に操作する」 |

**原則:** 業務ゾーンのナビにマスタ CRUD 画面を **同列で並べない**。保守ゾーンに入ったら、業務メニューと保守メニューを **混在させない**。

### 4.2 共通ルール

| # | ルール |
|---|--------|
| N-01 | グローバルナビは全ログイン済み画面で同一の3項目とする |

## 6. 保守モードの視覚仕様（ドラフト）

文言と合わせて「保守に入った」ことを伝えるための最低限のルール。

| 項目 | 業務ゾーン | マスタ保守ゾーン |
|------|------------|------------------|
| ページ背景 | 現行どおり（`--bg: #f4f1ea`） | 同左で可 |
| 保守バナー | なし | 全保守画面のヘッダー直下に固定表示 |
| バナー背景色 | — | 警告系（例：`#fff6e5` / 文字 `#8a5a00`） |
| バナーアイコン | — | 任意（⚙ または「保守」ラベル） |
| プライマリボタン色 | 現行アクセント（緑系） | 同左で可（変更必須ではない） |

**保守バナーの HTML 目安**

- `role="note"` または `class="maintenance-banner"`
- 保守ポータル（SCR-MT-00）にも、ポータルである旨の短い注意文を表示
