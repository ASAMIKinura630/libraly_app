図書館保守システム 概要 画面説明 DB(ERD)

DB(ERD) — エンティティ・リレーションシップ・ダイヤグラム

図書館保守システムで使用する全7テーブル（＋認証）の相関図

表記形式：ファイル名 / ユニークキー　フィールド　フィールド …（横並び）

libraly_app_category / id name 登録日時

1 : N → libraly_app（カテゴリID・削除時NULL）

libraly_app / id タイトル あらすじ 著者 カテゴリID 登録日時

1 : N → libraly_app_lending_history（図書ID）

libraly_app_faculty / id name 登録日時

1 : N → libraly_app_department（学部ID）

libraly_app_department / id (学部ID+学科名) 学部ID 学科名 登録日時

1 : N → libraly_app_student（学科ID）

libraly_app_student / id 学籍番号 氏名 学科ID 生年月日 登録日時

1 : N → libraly_app_lending_history（学籍番号）

libraly_app_staff / id 職員番号 メールアドレス 氏名 登録日時

1 : N → libraly_app_lending_history（貸出・返却処理者ID）

メール一致 ↔ auth.users（論理紐づけ）

auth.users / id メールアドレス （Supabase Auth・DB外部）
libraly_app_lending_history / id 図書ID 学籍番号 返却フラグ 返却予定日 返却日時 貸出日時 貸出処理者ID 返却処理者ID
ファイル名 テーブル名 id ユニークキー（PK・UNIQUE） 項目 フィールド FK 外部キー

テーブル一覧（全7）

  • libraly_app_category — 図書カテゴリマスタ
  • libraly_app — 図書マスタ
  • libraly_app_faculty — 学部マスタ
  • libraly_app_department — 学科マスタ
  • libraly_app_student — 学生マスタ
  • libraly_app_staff — 担当者マスタ
  • libraly_app_lending_history — 図書貸出履歴

主なリレーション

  • カテゴリ → 図書（1対多、削除時はカテゴリIDを空にする）
  • 学部 → 学科 → 学生（1対多、参照中は削除不可）
  • 図書・学生 → 貸出履歴（1対多、参照中は削除不可）
  • 担当者 → 貸出履歴（貸出・返却の処理者）
  • 認証ユーザー ↔ 担当者（メールアドレス一致で論理紐づけ）
