図書館保守システム 概要 画面説明 DB(ERD)

DB(ERD) — エンティティ・リレーションシップ・ダイヤグラム

各テーブルを「ファイル名 / ユニークキー　フィールド…」の横並びで表示

libraly_app_category / ID カテゴリ名 登録日時
│ 1 : N → 図書マスタ（カテゴリID）
libraly_app / ID カテゴリID タイトル あらすじ 著者 登録日時
│ 1 : N → 図書貸出履歴（図書ID）
libraly_app_lending_history / ID 図書ID 学籍番号 返却フラグ 返却予定日 返却日時 貸出日時 貸出処理者ID 返却処理者ID
│ 1 : N ← 学生マスタ（学籍番号）
libraly_app_student / 学籍番号 学科ID 氏名 生年月日 登録日時
│ 1 : N → 学科マスタ（学科ID）
libraly_app_department / ID 学部ID 学科名 登録日時
│ 1 : N → 学部マスタ（学部ID）
libraly_app_faculty / ID 学部名 登録日時
│ 1 : N → 図書貸出履歴（貸出処理者ID・返却処理者ID）
libraly_app_staff / 職員番号 氏名 メールアドレス 登録日時
┊ メールアドレス一致（論理参照）
auth.users / メールアドレス （Supabase Auth）
ユニークキー テーブルを一意に識別 外部キー 他テーブル参照 │ 1対多の関連

テーブル一覧（全7＋認証）

  • libraly_app_category — 図書カテゴリマスタ
  • libraly_app — 図書マスタ
  • libraly_app_faculty — 学部マスタ
  • libraly_app_department — 学科マスタ
  • libraly_app_student — 学生マスタ
  • libraly_app_staff — 担当者マスタ
  • libraly_app_lending_history — 図書貸出履歴
  • auth.users — 認証（DB外・論理参照）

主なリレーション

  • カテゴリ → 図書（1対多、削除時はカテゴリIDを空にする）
  • 学部 → 学科 → 学生（1対多、参照中は削除不可）
  • 図書・学生 → 貸出履歴（1対多、参照中は削除不可）
  • 担当者 → 貸出履歴（貸出・返却の処理者）
  • 認証ユーザー ↔ 担当者（メールアドレス一致で論理紐づけ）
